# portal-intl-archive

## 项目简介

本仓库用于归档与发布一系列独立 HTML 页面。这些页面通常作为信息门户、工具界面或静态展示页使用，不针对特定网站或域名。仓库本身不提供动态服务，仅作为静态资源的版本化存储与发布源。

## 目录说明

```
portal-intl-archive/
├── pages/          # 存放各独立 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── docs/           # 可选：额外文档或说明
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名通常反映其用途或内容主题。
- `assets/`：按需组织子目录，资源文件与页面文件保持相对路径引用。
- `docs/`：如需附加说明、设计稿或使用指南，可放置于此。

## 页面归档说明

- 所有页面均为纯前端静态页面，不依赖后端服务。
- 页面之间相互独立，无内部链接依赖（除非手动建立）。
- 归档的页面可能来自不同时期或用途，内容不保证彼此关联。
- 每个页面可单独通过浏览器打开，无需额外配置。

### 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中直接打开 `pages/` 下的 HTML 文件。
3. 或通过 GitHub Pages 等静态托管服务部署整个仓库（需自行配置）。

## 维护说明

- 本仓库仅用于归档，不承诺定期更新或修复。
- 如需新增页面，请将 HTML 文件放入 `pages/`，相关资源放入 `assets/`，并保持路径一致性。
- 若页面引用外部资源（如字体、库），建议尽量内联或本地化，以减少对外部服务的依赖。
- 欢迎提交 Issue 或 Pull Request 以修正明显错误或添加新的归档页面，但请确保内容符合仓库的归档性质。
